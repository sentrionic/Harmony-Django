Story API


Posting a Story

Request: POST

Headers:

Authorization: Token <token>

Url:

website/api/story/create

Params:

caption (text)

tags (text)

image (file)


Success Response:

{
    "caption": "this is new body \ncool.\nstuff. #wow",
    "tags": "#wow",
    "image": "/media/story/1/story_image.png",
    "date_published": "2019-07-09T15:19:33.095626Z",
    "username": "username"
}

List Story Posts (Pagination, Search and Ordering)

Request: GET

Headers:

Authorization: Token <token>

Url:

website/api/story/list/?search=<query>&ordering=-date_updated&page=1

Params:

query (this will search the title and body of blog posts for a match)

ordering (Can use: caption, tags, username or date_published)

page (10 results will be shown per page.)


Success Response:

{
    "count": <num_results>,
    "next": null,
    "previous": null,
    "results": [
        {
            "pk": "2",
            "slug": "abcdef",
            "caption": "gfhhfghfgh",
            "tags": "fghgfhfghf",
            "image": "/media/story/1/story_image.png",
            "date_published": "2019-07-05T21:43:56.452234Z",
            "username": "username",
            "likes": "2",
            "liked": "boolean",
            "image": "/media/account/1/pfp.png",
            "comments": "0"
        },
        ...
}

Alternatives:

Liked Posts

Url:

website/api/story/liked/?search=<query>&ordering=-date_updated&page=1

Returns a list of stories the user liked


Followed Posts

Url:

website/api/story/followed/?search=<query>&ordering=-date_updated&page=1

Returns a list of stories of accounts the user follows

Account API


Obtaining an Authentication Token

Request: POST

Url:

website/api/account/login

Params:

username (Enter your login email)

password (Your password)

No that's not a typo above. The param is username but you use your email.


Success Response:

{
    "response": "Successfully authenticated.",
    "pk": 1,
    "email": "email@example.com",
    "token": "27a46967801960de8734f65ba13b124aaea96ad9"
}
